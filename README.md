# 🚀 Odesa Wave — Roadmap розвитку продукту

> Муніципальний мобільний застосунок для Одеси (React Native + FastAPI + MongoDB)

## Пріоритизація

**Принцип**: максимальна цінність для одеситів при мінімальних регуляторних бар'єрах

---

## 1. 🏛 ДЕРЖАВНІ ІНТЕГРАЦІЇ

### 1.1 Дія API
- **API**: `integration.diia.gov.ua` — валідація, шеринг документів, авторизація
- **Складність**: 3/5
- **Бізнес-цінність**: Верифікація віку/особи без передачі персональних даних
- **Обмеження**: Потрібна реєстрація бізнесу + погодження Мінцифри (1-3 місяці)
- **Рекомендація**: Почати з QR-валідації, потім — повна інтеграція

### 1.2 BankID НБУ
- **API**: `id.gov.ua` — Система BankID
- **Складність**: 2/5
- **Бізнес-цінність**: Верифікація особи; 109 млн ідентифікацій у 2025 (+25% р/р)
- **Посилання**: https://bank.gov.ua/ua/news/all/sistema-bankid-nbu-u-2025-rotsi-zrostaye-popit-ta-dovira-koristuvachiv
- **Рекомендація**: ✅ ПРІОРИТЕТ — найпростіший для швидкої верифікації

### 1.3 Трембіта
- **API**: `trembita.gov.ua` — 10+ млрд транзакцій; Трембіта 2.0 у пілоті
- **Складність**: 5/5
- **Бізнес-цінність**: Доступ до Держреєстрів (ЄДР, нерухомість, МВС)
- **Обмеження**: Складна інтеграція; потрібні КЕП та погодження держоргану
- **Рекомендація**: Не для MVP — тільки після запиту від мерії

### 1.4 ЦНАП Одеса
- **URL**: `odesacnap.gov.ua`
- **Складність**: 3/5
- **Рекомендація**: Почати з WebView → пряма інтеграція після офіційного запиту

### 1.5 ДІЯ.Підпис
- **API**: Вбудований у мобільний застосунок Дія
- **Складність**: 3/5
- **Рекомендація**: Запит через QR-код + deep link на Дію

## 7. 🌍 МАСШТАБУВАННЯ

### 7.1 Multi-city архітектура

```
Backend (FastAPI)
    │
    ├─ MongoDB (Odesa)
    ├─ MongoDB (Lviv)
    └─ MongoDB (Kharkiv)
          │
    City-aware API layer
    { city: "odesa", lang: "ua" }
```

### 7.2 Пріоритетні міста

| Місто | Потенціал | Причина |
|---|---|---|
| Львів | Високий | 700K+, туризм, мерія active |
| Харків | Високий | 1.4M, метро |
| Дніпро | Середній | 1M, транзит |

### 7.3 Зміни в архітектурі
- `db.cities` + `city_id` у кожному запиті
- Feature flags для city-specific функцій
- Окремі GTFS для кожного міста

## 8. 🤖 AI / ML

### 8.1 "Дядя Жора" — RAG
- **Технологія**: LangChain + Qdrant + Ukrainian BERT
- **Складність**: 3/5
- **Рекомендація**: ✅ ПРІОРИТЕТ 2 — спочатку knowledge base

### 8.2 Предиктивний аналіз відключень
- **Model**: LightGBM / Prophet
- **Data**: "Де Світло" crowd-sourced + погода
- **Складність**: 4/5

### 8.3 Комп'ютерний зір (SentryNode)
- **Model**: YOLOv8 / TensorFlow Lite
- **Складність**: 4/5
